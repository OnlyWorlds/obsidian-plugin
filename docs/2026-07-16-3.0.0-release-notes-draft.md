# 3.0.0 — draft release notes + release checklist

**Created**: 2026-07-16 (Skeld). **Status**: DRAFT — release gates on Captain's word. Branch `phase-bc`, not merged, not pushed. Real-vault smoke passed across the session (download → frontmatter, clickable links, migration audited clean, folder→Atlas, template-free download, read-key upload refusal). Version raised 2.4.0 → **3.0.0** (Captain's call): the note format changed, the span era is retired, and the folder bridge is a new interop surface — a major, not a point release.

## The story (one line)
**The frontmatter era: your world as readable Obsidian notes — fields in Properties, links clickable and in the graph — with a portable folder bridge to the wider ecosystem.**

## What's new in 3.0.0

**Frontmatter format.** Element fields now live in the note's Properties (YAML frontmatter). Link fields are clickable `[[Name]]` wikilinks — they appear in Obsidian's graph and backlinks, and resolve by filename so a moved vault keeps working with no plugin needed. Empty fields are omitted; `id`/`image_url` sit at the bottom, name first. Extension/custom fields (`atlas_*`/`shadow_*`/`x_*`) round-trip verbatim (the silent-drop bug of older versions is gone).

**Migrate command.** `Migrate world notes to frontmatter` converts an existing world. Backs up every note first (`OW-backup-<world>-<timestamp>/`), aborts if the backup fails, idempotent. Old span-format notes still READ correctly without migrating, so updating never breaks an existing vault.

**Folder bridge (open format, no account needed).** `Export as OnlyWorlds folder` writes a portable OnlyWorlds folder — point it at your Atlas root to open the world in Atlas directly, or the vault default and move it. `Import OnlyWorlds folder` reads one back into frontmatter notes (never overwrites existing, never merges two worlds). The plugin is now a full implementation of the OnlyWorlds folder format.

**Read-only keys need no PIN.** Downloading with a read-capability key (`ow_r_`) or a demo key no longer asks for a PIN — the PIN field greys out. Open a world someone shared with you without inventing a PIN you don't have. (Write keys and private-world write keys still require the PIN — unchanged.)

**Leaner vaults.** New/downloaded/pasted worlds no longer fetch or write the old `PluginFiles/Templates` + `PluginFiles/Handlebars` template sets — nothing read them anymore. No more per-world GitHub fetch. (Existing vaults keep their old template folders harmlessly; delete them if you like.)

**Every command is frontmatter-native.** Create, Download, Paste, Copy, Save, Link, Migrate, Export/Import folder — all write/read the new format. `Paste World` (previously still wrote the old span format) now writes frontmatter and round-trips with `Copy World`.

**Under the hood.** All sync on the v2 API (from 2.3.0): incremental `/changes` download, safe upload sweep (create + update, never delete), read-before-PATCH saves.

## Known limitations (honest)
- `Validate World` only checks the legacy span format; on a migrated world it now SKIPS frontmatter notes rather than showing false errors. It's read-only, so it's safe. A frontmatter-aware validator is planned — the one command not yet frontmatter-native.
