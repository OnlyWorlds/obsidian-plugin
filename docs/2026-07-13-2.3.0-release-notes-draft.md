## Release notes (draft — GitHub release body)

### OnlyWorlds v2 API under the hood

Every sync path now speaks the OnlyWorlds v2 API. No note-format changes — your vault works exactly as before, but sync is faster, safer, and more honest.

- **Download World is incremental.** The first download pulls everything; every re-download pulls only what changed since (powered by the v2 change feed). Re-downloading a big world takes seconds.
- **Upload World no longer full-replaces.** Elements are created or updated individually — and elements that exist on the server but not in your vault are **reported, never deleted**. The silent-delete risk of the old bulk upload is gone.
- **New worlds with cross-links upload reliably.** First pushes go through the v2 bulk endpoint, which resolves link ordering server-side (mutually-linked elements can't fail on upload order anymore).
- **Deletes are surfaced, not silent.** If elements were deleted on the server, downloading tells you which — your notes are never auto-removed.
- **Clearer errors.** API errors now show the server's actual message, with a link to the error docs.

Under the hood: the legacy worldsync endpoints are fully retired; auth rides API-Key/API-Pin headers everywhere; the SDK remains the schema authority.

## Added during the smoke-test cycle (2026-07-13 afternoon)
