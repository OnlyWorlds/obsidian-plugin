# 2.4.0 — draft release notes + release checklist

**Created**: 2026-07-16 (Skeld). **Status**: DRAFT — release gates on Captain's word. Branch `phase-bc`, not merged, not pushed. Captain's real-vault smoke already passed (download → frontmatter, clickable links, migration on MessWorld audited clean, folder→Atlas confirmed). Draft the version + framing, then release.

## The story (one line)
**Your world, readable and connected: elements are now Obsidian Properties, and links are clickable, graph-aware `[[wikilinks]]`.**

## What's new in 2.4.0

**Frontmatter format.** Element fields now live in the note's Properties (YAML frontmatter) instead of inline `<span>` lines. Link fields are clickable `[[Name]]` wikilinks — they show up in the graph and backlinks, and resolve by filename so a moved vault keeps working with no plugin needed. Notes read as prose with structured data attached.

**Migrate command.** `Migrate world notes to frontmatter` converts an existing world. Backs up every note first (`OW-backup-<world>-<timestamp>/`), aborts if the backup fails, idempotent (safe to run twice). Old span-format notes still READ correctly without migrating, so updating never breaks an existing vault.

**Folder bridge (open format, no account needed).** `Export as OnlyWorlds folder` writes a portable OnlyWorlds folder — point it at your Atlas root to open the world in Atlas directly, or the vault default and move it yourself. `Import OnlyWorlds folder` reads one back into frontmatter notes (never overwrites existing, never merges two worlds). Makes the plugin a full implementation of the OnlyWorlds folder format.

**Cleaner notes.** Empty fields are omitted; `id`/`image_url` sit at the bottom of Properties, name first. Extension fields (`atlas_*`/`shadow_*`/`x_*` custom fields) round-trip verbatim — the silent-drop bug of older versions is gone.

**Under the hood.** All sync stays on the v2 API (from 2.3.0): incremental `/changes` download, safe upload sweep, read-before-PATCH saves.

## Known limitations (honest, deferred to a future release)
- `Validate World` only checks the legacy span format; on a migrated world it now SKIPS frontmatter notes rather than showing false errors. A frontmatter-aware validator is planned.
- `Paste World from Clipboard` still writes span-format notes (a self-contained import path; the notes remain readable and migratable).
